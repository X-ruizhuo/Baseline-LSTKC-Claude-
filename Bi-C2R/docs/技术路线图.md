# 技术路线图：Baseline vs 本项目

## 一、Baseline (Bi-C2R) 技术路线

### 训练流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                  Baseline (Bi-C2R) 训练流程                      │
└─────────────────────────────────────────────────────────────────┘

阶段0: 第一个数据集 (Market-1501 或 DukeMTMC)
├─ 初始化 ResNet-50
├─ 训练分类器 (num_classes_0)
├─ 优化: Loss_ce + Loss_tp
└─ 保存: model_0

阶段1-4: 后续数据集 (增量学习)
│
├─ 步骤1: 准备阶段
│   ├─ 加载上一阶段模型 → model_old (冻结)
│   ├─ 深拷贝当前模型 → model_new (可训练)
│   ├─ 扩展分类器: Linear(2048, add_num + num_classes_new)
│   └─ 初始化新类别中心 (用新数据集特征均值)
│
├─ 步骤2: 训练循环 (每个epoch)
│   │
│   ├─ 2.1 前向传播
│   │   ├─ model_new(x) → f_new, cls_new
│   │   └─ model_old(x) → f_old, cls_old (no_grad)
│   │
│   ├─ 2.2 基础损失
│   │   ├─ Loss_ce = CrossEntropy(cls_new, targets)
│   │   └─ Loss_tp = TripletLoss(f_new, targets)
│   │
│   ├─ 2.3 防遗忘损失
│   │   ├─ Affinity_new = softmax(f_new @ f_new.T / 0.1)
│   │   ├─ Affinity_old = softmax(f_old @ f_old.T / 0.1)
│   │   ├─ 构建目标矩阵 (保持正确预测 + 修正错误预测)
│   │   └─ Loss_KL = KLDiv(log(Affinity_new), Target)
│   │
│   ├─ 2.4 双向转换损失
│   │   ├─ f_old→new = TransNet_1(f_old)
│   │   ├─ f_new→old = TransNet_2(f_new)
│   │   ├─ Loss_trans = MSE(f_old→new, f_new) + MSE(f_new→old, f_old)
│   │   ├─ Loss_anti = KL_compatibility(f_old, f_old→new) + KL_compatibility(f_new, f_new→old)
│   │   ├─ Loss_discri = KD(cls_trans_old, cls_old) + KD(cls_trans_new, cls_new)
│   │   └─ Loss_transx = 1 - Cosine(Δf_new, Δf_trans_old) + 1 - Cosine(Δf_old, Δf_trans_new)
│   │
│   ├─ 2.5 总损失
│   │   └─ Loss_total = Loss_ce + Loss_tp + Loss_KL + 
│   │                   100*Loss_trans + 1*Loss_anti + 
│   │                   0.007*Loss_discri + 0.0005*Loss_transx
│   │
│   └─ 2.6 反向传播
│       └─ optimizer.step()
│
├─ 步骤3: 自适应融合
│   ├─ 提取新旧模型在当前数据集上的特征
│   ├─ 计算亲和矩阵差异: Difference = |Affinity_new - Affinity_old|
│   ├─ 计算融合权重: alpha = 1 - Difference
│   └─ 融合模型: model_final = alpha * model_new + (1-alpha) * model_old
│
├─ 步骤4: 特征库更新
│   ├─ 提取当前数据集gallery特征 → 保存
│   └─ 转换旧数据集特征: f_old_gallery → TransNet → f_old_updated
│
└─ 步骤5: 评估
    ├─ Seen datasets: 使用转换后的特征
    └─ Unseen datasets: 直接使用当前模型

【关键技术点】
✓ 双向转换网络保持新旧特征空间兼容
✓ 基于亲和矩阵的自适应融合
✓ 多损失函数协同优化
✗ 新旧知识混合，容易冲突
✗ 无质量评估，可能负迁移
```

## 二、本项目 (Bi-C2R + LSTKC++) 技术路线

### 训练流程图

```
┌═════════════════════════════════════════════════════════════════┐
║            本项目 (Bi-C2R + LSTKC++) 增强训练流程                ║
╚═════════════════════════════════════════════════════════════════╝

阶段0: 第一个数据集 (与Baseline相同)
├─ 初始化 ResNet-50
├─ 训练分类器 (num_classes_0)
├─ 优化: Loss_ce + Loss_tp
└─ 保存: model_0

阶段1-4: 后续数据集 (LSTKC++增强增量学习)
│
├─ 步骤1: 准备阶段 (与Baseline相同)
│   ├─ 加载上一阶段模型 → model_old (冻结)
│   ├─ 深拷贝当前模型 → model_new (可训练)
│   ├─ 扩展分类器: Linear(2048, add_num + num_classes_new)
│   ├─ 初始化新类别中心
│   └─ 【新增】初始化知识分解/过滤模块
│
├─ 步骤2: 训练循环 (每个epoch) - 增强版
│   │
│   ├─ 2.1 前向传播 (与Baseline相同)
│   │   ├─ model_new(x) → f_new, cls_new
│   │   └─ model_old(x) → f_old, cls_old (no_grad)
│   │
│   ├─ 2.2 基础损失 (与Baseline相同)
│   │   ├─ Loss_ce = CrossEntropy(cls_new, targets)
│   │   └─ Loss_tp = TripletLoss(f_new, targets)
│   │
│   ├─ 2.3 防遗忘损失 (与Baseline相同)
│   │   ├─ Affinity_new = softmax(f_new @ f_new.T / 0.1)
│   │   ├─ Affinity_old = softmax(f_old @ f_old.T / 0.1)
│   │   └─ Loss_KL = KLDiv(log(Affinity_new), Target)
│   │
│   ├─ 【新增】2.4 知识分解阶段
│   │   │
│   │   ├─ 分解旧模型特征:
│   │   │   ├─ LT_old = LongTermExtractor(f_old)      # 通用知识
│   │   │   ├─ ST_old = ShortTermExtractor(f_old)     # 任务特定知识
│   │   │   ├─ gate_old = GateNetwork(f_old)          # 门控权重
│   │   │   └─ f_old_decomp = gate_old * LT_old + (1-gate_old) * ST_old
│   │   │
│   │   └─ 分解新模型特征:
│   │       ├─ LT_new = LongTermExtractor(f_new)
│   │       ├─ ST_new = ShortTermExtractor(f_new)
│   │       ├─ gate_new = GateNetwork(f_new)
│   │       └─ f_new_decomp = gate_new * LT_new + (1-gate_new) * ST_new
│   │
│   ├─ 【新增】2.5 知识过滤阶段
│   │   ├─ 拼接特征: combined = [f_old, f_new]  # 4096维
│   │   ├─ 质量评估: quality_score = QualityEstimator(combined)
│   │   ├─ 生成掩码: mask = (quality_score > 0.6).float()
│   │   ├─ 过滤特征: f_old_filtered = f_old * quality_score * mask + 
│   │   │                             f_old * (1-mask) * 0.5
│   │   └─ 计算权重: quality_weight = quality_score.mean()
│   │
│   ├─ 2.6 双向转换损失 (增强版)
│   │   ├─ 使用分解后的特征:
│   │   │   ├─ f_old_decomp→new = TransNet_1(f_old_decomp)
│   │   │   └─ f_new_decomp→old = TransNet_2(f_new_decomp)
│   │   │
│   │   ├─ 【增强】质量加权的转换损失:
│   │   │   └─ Loss_trans = quality_weight * [
│   │   │         MSE(f_old_decomp→new, f_new) + 
│   │   │         MSE(f_new_decomp→old, f_old)
│   │   │       ]
│   │   │
│   │   ├─ Loss_anti (与Baseline相同)
│   │   ├─ Loss_discri (与Baseline相同)
│   │   └─ Loss_transx (与Baseline相同)
│   │
│   ├─ 【新增】2.7 长短期知识损失
│   │   ├─ Loss_long_term = MSE(LT_new, LT_old.detach())
│   │   │   目的: 保持长期通用知识稳定
│   │   │   权重: 0.5
│   │   │
│   │   └─ Loss_short_term = MSE(ST_new, f_new)
│   │       目的: 短期知识快速适应新任务
│   │       权重: 0.3
│   │
│   ├─ 2.8 总损失 (增强版)
│   │   └─ Loss_total = Loss_ce + Loss_tp + Loss_KL + 
│   │                   quality_weight * 100 * Loss_trans + 
│   │                   1 * Loss_anti + 
│   │                   0.007 * Loss_discri + 
│   │                   0.0005 * Loss_transx +
│   │                   0.5 * Loss_long_term +      【新增】
│   │                   0.3 * Loss_short_term        【新增】
│   │
│   └─ 2.9 反向传播 (混合精度优化)
│       ├─ 【新增】使用FP16自动混合精度
│       ├─ scaler.scale(loss / gradient_accumulation_steps).backward()
│       ├─ 【新增】梯度累积 (每2步更新一次)
│       └─ if (i+1) % 2 == 0: scaler.step(optimizer)
│
├─ 步骤3: 增强的自适应融合
│   ├─ 基础alpha计算 (与Baseline相同):
│   │   ├─ Difference = |Affinity_new - Affinity_old|
│   │   └─ alpha_base = 1 - Difference
│   │
│   ├─ 【新增】长短期稳定性评估:
│   │   ├─ stability = StabilityEstimator(LT_old)
│   │   ├─ importance = ImportanceEstimator(ST_old)
│   │   └─ alpha_enhanced = (1 - Difference) * stability + 
│   │                       Difference * (1 - importance)
│   │
│   ├─ 限制范围: alpha_final = clamp(alpha_enhanced, 0.3, 0.9)
│   └─ 融合模型: model_final = alpha_final * model_new + 
│                              (1-alpha_final) * model_old
│
├─ 步骤4: 特征库更新 (与Baseline相同)
│   ├─ 提取当前数据集gallery特征 → 保存
│   └─ 转换旧数据集特征: f_old_gallery → TransNet → f_old_updated
│
└─ 步骤5: 评估 (与Baseline相同)
    ├─ Seen datasets: 使用转换后的特征
    └─ Unseen datasets: 直接使用当前模型

【核心创新点】
✓ 显式长短期知识分解 (LT/ST)
✓ 知识质量过滤 (quality_score)
✓ 质量加权的损失函数
✓ 增强的自适应融合 (考虑稳定性)
✓ 混合精度训练 (FP16)
✓ 梯度累积优化
```

## 三、关键技术对比表

### 3.1 特征处理流程对比

| 阶段 | Baseline | 本项目 (LSTKC++) |
|------|----------|------------------|
| **特征提取** | ResNet → f (2048维) | ResNet → f (2048维) |
| **特征分解** | ❌ 无 | ✅ f → LT (通用) + ST (专用) |
| **质量评估** | ❌ 无 | ✅ QualityEstimator([f_old, f_new]) |
| **特征转换** | TransNet(f) | TransNet(f_decomp) |
| **特征融合** | 基于alpha直接融合 | 基于alpha + 质量权重融合 |

### 3.2 损失函数对比

| 损失类型 | Baseline | 本项目 | 权重变化 |
|---------|----------|--------|---------|
| Loss_ce | ✓ | ✓ | 1.0 (相同) |
| Loss_tp | ✓ | ✓ | 1.0 (相同) |
| Loss_KL | ✓ | ✓ | 1.0 (相同) |
| Loss_trans | ✓ | ✓ | 100 → 100 × quality_weight |
| Loss_anti | ✓ | ✓ | 1.0 (相同) |
| Loss_discri | ✓ | ✓ | 0.007 (相同) |
| Loss_transx | ✓ | ✓ | 0.0005 (相同) |
| Loss_long_term | ❌ | ✅ | 0.5 (新增) |
| Loss_short_term | ❌ | ✅ | 0.3 (新增) |
| **总计** | **6个损失** | **8个损失** | **+2个知识损失** |

### 3.3 Alpha计算对比

**Baseline:**
```python
Affinity_new = softmax(f_new @ f_new.T)
Affinity_old = softmax(f_old @ f_old.T)
Difference = |Affinity_new - Affinity_old|.mean()
alpha = 1 - Difference
```

**本项目 (LSTKC++):**
```python
# 基础计算 (继承Baseline)
Difference = |Affinity_new - Affinity_old|.mean()
alpha_base = 1 - Difference

# 增强计算 (新增)
stability = StabilityEstimator(LT_old).mean()
importance = ImportanceEstimator(ST_old).mean()
alpha_enhanced = (1 - Difference) * stability + 
                 Difference * (1 - importance)
alpha_final = clamp(alpha_enhanced, 0.3, 0.9)
```

### 3.4 训练优化对比

| 优化技术 | Baseline | 本项目 |
|---------|----------|--------|
| 精度 | FP32 | FP16 (混合精度) |
| Batch Size | 64 | 48 |
| 梯度累积 | 1步 | 2步 |
| 有效Batch | 64 | 48×2=96 |
| 显存占用 | ~32GB | ~21GB |
| 显存节省 | - | 34% |

## 七、运行命令对比

### Baseline运行

```bash
# Setting 1
bash run1.sh

# 等价于:
CUDA_VISIBLE_DEVICES=0,1 python continual_train.py \
    --logs-dir logs-baseline/ \
    --setting 1 \
    --batch-size 64 \
    --epochs0 80 \
    --epochs 60
```

### 本项目运行

```bash
# LSTKC++增强版
bash run_lstkc.sh

# 等价于:
CUDA_VISIBLE_DEVICES=0,1 python continual_train.py \
    --logs-dir logs-lstkc/ \
    --setting 1 \
    --enable_lstkc \              # 启用LSTKC++
    --use_amp \                   # 混合精度
    --gradient_accumulation_steps 2 \  # 梯度累积
    --batch-size 48 \             # 优化后batch size
    --weight_long_term 0.5 \      # 长期知识权重
    --weight_short_term 0.3 \     # 短期知识权重
    --epochs0 80 \
    --epochs 60
```
