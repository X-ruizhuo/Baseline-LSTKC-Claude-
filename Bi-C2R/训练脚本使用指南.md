## 核心参数说明

### 基础训练参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--setting` | 1 | 数据集顺序 (1或2) |
| `--epochs0` | 80 | 第一个数据集训练轮数 |
| `--epochs` | 60 | 后续数据集训练轮数 |
| `--batch_size` | 64 | 批次大小 |
| `--lr` | 0.008 | 学习率 |
| `--optimizer` | SGD | 优化器 |

### BI-C2R 参数
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--weight_trans` | 100 | 转换损失权重 |
| `--weight_anti` | 1 | 反遗忘损失权重 |
| `--weight_discri` | 0.007 | 判别损失权重 |
| `--weight_transx` | 0.0005 | 交叉转换损失权重 |
| `--AF_weight` | 1.0 | 亲和力矩阵权重 |

### LSTKC++ 参数 (新增)
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--use_lstkc` | True | 是否启用 LSTKC++ |
| `--decouple_epochs` | 5 | 知识解耦训练轮数 |
| `--decouple_lr` | 0.001 | 知识解耦学习率 |
| `--long_align_weight` | 1.0 | 长期对齐损失权重 |
| `--long_relation_weight` | 2.0 | 长期关系损失权重 (强约束) |
| `--short_adapt_weight` | 0.5 | 短期适应损失权重 (弱约束) |
| `--short_relation_weight` | 0.5 | 短期关系损失权重 |

## 训练流程

### LSTKC++ 融合训练流程

```
数据集 0: Market1501 (第一个数据集)
├─ 训练 80 epochs
└─ 保存模型和特征

数据集 1: DukeMTMC
├─ 知识解耦 (5 epochs)
│  ├─ 训练长期模型 (域不变特征)
│  └─ 训练短期模型 (场景特异特征)
├─ 新任务训练 (60 epochs)
│  └─ 使用长短期知识融合损失
└─ 保存模型和特征

数据集 2: CUHK03
├─ 知识解耦 (5 epochs)
├─ 新任务训练 (60 epochs)
└─ 保存模型和特征

数据集 3: MSMT17
├─ 知识解耦 (5 epochs)
├─ 新任务训练 (60 epochs)
└─ 保存模型和特征
```

## 故障排查

### 常见问题

#### 1. 数据路径错误
```bash
# 修改脚本中的数据路径
--data-dir /your/path/to/PRID
```

#### 2. GPU 内存不足
```bash
# 减小 batch size
--batch_size 32  # 从 64 改为 32
```

#### 3. CUDA 设备选择
```bash
# 使用不同的 GPU
CUDA_VISIBLE_DEVICES=1 bash run_lstkc_setting1.sh
```

#### 4. 禁用 LSTKC++ (回退到 Baseline)
```bash
# 在脚本中修改
--use_lstkc False
```

## 自定义配置

### 修改超参数

编辑脚本文件,调整以下参数:

```bash
# 知识解耦配置
--decouple_epochs 10      # 增加解耦训练轮数
--decouple_lr 0.0005      # 降低解耦学习率

# LSTKC++ 权重配置
--long_relation_weight 3.0   # 增强长期约束
--short_adapt_weight 0.3     # 减弱短期约束
```

### 添加新数据集

修改 [continual_train.py](continual_train.py) 中的数据集列表:

```python
if 1 == args.setting:
    training_set = ['market1501', 'dukemtmc', 'cuhk03', 'msmt17', 'your_dataset']
```
