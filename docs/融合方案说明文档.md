# LSTKC++ 与 Bi-C2R Baseline 融合方案说明文档

## 一、参考文献核心框架解析

### 1.1 论文概述

**论文**：Long Short-Term Knowledge Decomposition and Consolidation for Lifelong Person Re-Identification  
**发表**：IEEE TPAMI, Vol. 47, No. 9, September 2025  
**作者**：Kunlun Xu, Zichen Liu, Xu Zou, Yuxin Peng, Jiahuan Zhou（北京大学）

**核心问题**：现有 exemplar-free LReID 方法直接蒸馏旧模型知识，存在两个缺陷：
1. 旧模型预测中包含错误知识，无选择地蒸馏会污染新知识学习
2. 部分关键知识因无旧数据而永久丢失

---

### 1.2 LSTKC（基础版）框架

LSTKC 包含两个核心模块：

#### 模块 Mr：Rectification-based Short-Term Knowledge Transfer（R-STKT）

**目标**：识别并纠正旧模型预测中的错误知识，再蒸馏给新模型。

**数据流**：
```
新数据批次 I = {(x_i, y_i)}^B
        ↓
新模型 Θ_t^s 提取特征 V_t^s ∈ R^{B×d}
        ↓ L2归一化
相似度矩阵 S_t^s = V̄_t^s · (V̄_t^s)^T  ∈ R^{B×B}
        ↓ softmax(·/τ), τ=0.1
关系矩阵 R_t^s ∈ R^{B×B}   ← 新模型知识表示

旧模型 Θ_{t-1} 同样处理 I → R_{t-1} ∈ R^{B×B}  ← 旧模型知识表示
        ↓
【短期知识纠正 STKR】
  对 R_{t-1} 的每一行 i：
    sn = min{R_{t-1,ij}} s.t. y_i = y_j   (同ID最小亲和度 → 负阈值)
    sp = max{R_{t-1,ij}} s.t. y_i ≠ y_j   (异ID最大亲和度 → 正阈值)
  纠正规则：
    R̃_{t-1,ij} = max(R_{t-1,ij}, sp)  if y_i = y_j  (同ID分数不低于sp)
    R̃_{t-1,ij} = min(R_{t-1,ij}, sn)  if y_i ≠ y_j  (异ID分数不高于sn)
  → 对每行做 L1 归一化
        ↓
【纠正知识蒸馏】
  L_KL = (1/B) Σ_i KL(R̃_{t-1,i} || R_t^s_i)
  总损失：L = L_Base + λ·L_KL
```

#### 模块 Me：Estimation-based Long-Term Knowledge Consolidation（E-LTKC）

**目标**：训练结束后，通过估计新旧模型知识差异，自适应融合模型参数。

**数据流**：
```
用 Θ_{t-1} 和 Θ_t^s 分别处理全部训练数据 D_t
        ↓
得到关系矩阵 R_{t-1} 和 R_t^s ∈ R^{n_t × n_t}
        ↓
逐元素绝对差：ΔR_t = |R_{t-1} - R_t^s|
        ↓
知识差异标量：δ_t = (1/n_t) Σ_i Σ_j (ΔR_t)_{ij}   ← 公式(9)
        ↓
模型融合：Θ_t = (1 - δ_t)·Θ_t^s + δ_t·Θ_{t-1}
  (δ_t 越大 → 旧模型权重越高 → 更保守，防遗忘)
```

### 1.3 LSTKC++（完整版）框架

在 LSTKC 基础上，将单个旧模型分解为**长期模型 Θ_t^l**（多域知识）和**短期模型 Θ_{t-1}^s**（最新域知识），新增三个模块：

#### 模块 M*_r：Complementary Short-Term Transfer（互补短期迁移）

**目标**：利用长短期两个旧模型的互补知识进行更精准的纠正。

**数据流**：
```
Θ_{t-1}^s → R_{t-1}^s    (短期旧模型关系矩阵)
Θ_{t-1}^l → R_{t-1}^l    (长期旧模型关系矩阵)
        ↓ 分别用 STKR 纠正
R̃_{t-1}^s,  R̃_{t-1}^l
        ↓ 互补融合策略 C-STKR：
  对每个元素 (i,j)：
    情况1：两模型都正确或都错误 → 取平均 (R̃^s + R̃^l) / 2
    情况2：一个正确一个错误     → 只保留正确的那个
  → 最终纠正矩阵 R̃_{t-1}
        ↓
L_KL = (1/B) Σ_i KL(R̃_{t-1,i} || R_t^s_i)
总损失：L = L_Base + λ·L_KL  (与 LSTKC 相同形式)
```

#### 模块 M*_o：Old Knowledge Backward Rectification（旧知识反向纠正）

**目标**：用新数据作为无偏参考，搜索最优融合权重，更新长期模型。

**算法流程（Algorithm 1）**：
```
输入：新数据 D_t，长期旧模型 Θ_{t-1}^l，短期旧模型 Θ_{t-1}^s
输出：更新后的长期模型 Θ_t^l

best_mAP = 0, α* = 0
for α ∈ {0, 0.1, 0.2, ..., 1.0}:
    Θ_fused = (1-α)·Θ_{t-1}^s + α·Θ_{t-1}^l
    F = 提取 D_t 中所有图像的特征（用 Θ_fused）
    mAP = 计算 F 的自排序 mAP（F 同时作为 query 和 gallery）
    if mAP > best_mAP:
        α* = α, best_mAP = mAP

Θ_t^l = (1 - α*)·Θ_{t-1}^s + α*·Θ_{t-1}^l
```

#### 模块 M*_e：Relation Similarity-guided Long-Term Consolidation（关系相似度引导的长期整合）

**目标**：改进 δ_t 的计算方式，解决公式(9)在大域间隔时 δ_t > 1 的问题。

**改进的 δ_t 计算（公式15）**：
```
用 Θ_t^l 和 Θ_t^s 处理训练数据 → R_t^l, R_t^s ∈ R^{n_t × n_t}

δ_t = 1 - (1/n_t) Σ_i <(R_t^s)_i, (R_t^l)_i>
  (点积相似度，保证 δ_t ∈ [0,1])

最终推理模型：Θ_t = (1 - δ_t)·Θ_t^s + δ_t·Θ_t^l
```

**公式(9) vs 公式(15) 的关键区别**：
- 公式(9)：绝对差之和，理论上 δ_t ≤ 2，域间隔大时可能 > 1，导致新模型权重为负
- 公式(15)：点积相似度，严格保证 δ_t ∈ [0,1]，更稳定

#### LSTKC++ 完整训练流程（Algorithm 2）：
```
t=1: Θ_1^s = Mb(Θ_0, D_1);  Θ_1 = Θ_1^s
t=2: Θ_2^s = Mr(Θ_1, D_2);  Θ_2^l = Θ_1^s;  Θ_2 = M*_e(Θ_2^l, Θ_2^s, D_2)
t≥3: Θ_t^s = M*_r(Θ_{t-1}^l, Θ_{t-1}^s, D_t)
     Θ_t^l = M*_o(Θ_{t-1}^l, Θ_{t-1}^s)
     Θ_t   = M*_e(Θ_t^l, Θ_t^s, D_t)
```

## 二、Baseline（Bi-C2R）与 LSTKC++ 的模块对比

| 维度 | Bi-C2R Baseline | LSTKC++ |
|------|----------------|---------|
| 旧模型数量 | 1个（单一旧模型） | 2个（长期 Θ^l + 短期 Θ^s） |
| 知识蒸馏方式 | 亲和矩阵 KL 散度（含 TP/FP/TN/FN 加权软目标） | 纠正后亲和矩阵 KL 散度（STKR/C-STKR） |
| 知识纠正 | 无（直接蒸馏） | 有（阈值挖掘 + 阈值纠正） |
| 模型融合权重 | `alpha = 1 - |Affin_new - Affin_old|`（绝对差，可能越界） | `δ_t = 1 - dot(R^s, R^l)`（点积相似度，严格[0,1]） |
| 融合对象 | new model ↔ old model | new model ↔ long-term model |
| 长期模型更新 | 无（每步直接用上一步推理模型） | M*_o 反向搜索最优 α 更新长期模型 |
| 额外可学习模块 | TransNet_adaptive（2个，双向特征变换） | 无额外可学习模块 |
| 特征变换损失 | L_trans(MSE) + L_cr + L_discri + L_transx | 无 |
| 损失函数总数 | 7个（CE+Triplet+KL+MSE+CR+Discri+TransX） | 2个（L_Base + λ·L_KL） |
| 超参数复杂度 | 高（weight_trans=100, weight_anti=1, weight_discri=0.007, weight_transx=0.0005） | 低（λ=1, τ=0.1, γ=1） |
| 特征重索引 | 有（trans_feat，用 TransNet 更新旧 gallery 特征） | 无 |
| 推理模型 | 融合后的单一模型 | Θ_t（长短期融合） |

## 三、融合方案设计

### 3.1 融合思路

LSTKC++ 的核心贡献是**知识纠正**和**长短期模型分解**，而 Bi-C2R 的核心贡献是**双向特征变换网络（TransNet）**和**特征重索引**。两者在以下维度上互补：

- LSTKC++ 解决了"蒸馏错误知识"的问题，但没有特征空间对齐机制
- Bi-C2R 有强大的特征变换对齐，但知识蒸馏没有纠正机制，且 δ 计算可能越界

**融合目标**：用 LSTKC++ 的知识纠正和长短期分解替换/增强 Bi-C2R 的蒸馏模块，同时保留 Bi-C2R 的特征变换优势。

---

### 3.2 方案一：最小替换方案（推荐优先验证）

**替换内容**：仅替换 `cal_KL` 模块和 `get_adaptive_alpha` 模块，保留其余所有结构。

#### 替换点 1：`trainer.py` 中的 `cal_KL` → STKR 纠正蒸馏

**当前代码**（`reid/trainer.py:207-237`）：

**替换为**：STKR 纠正策略
```python
def cal_KL_rectified(self, R_new, R_old, targets):
    """
    用 LSTKC++ 的 STKR 策略纠正 R_old，再蒸馏给 R_new
    R_new, R_old: [B, B] 亲和矩阵（已经过 softmax）
    targets: [B] 身份标签
    """
    B = targets.shape[0]
    R_old_rectified = R_old.clone()

    for i in range(B):
        same_id_mask = (targets == targets[i])       # 同ID掩码
        diff_id_mask = ~same_id_mask                 # 异ID掩码
        same_id_mask[i] = False                      # 排除自身

        if same_id_mask.sum() > 0 and diff_id_mask.sum() > 0:
            sn = R_old[i][same_id_mask].min()        # 同ID最小值 → 负阈值
            sp = R_old[i][diff_id_mask].max()        # 异ID最大值 → 正阈值

            # 纠正同ID分数：不低于 sp
            intra_mask = same_id_mask
            R_old_rectified[i][intra_mask] = torch.maximum(R_old[i][intra_mask], sp)
            # 纠正异ID分数：不高于 sn
            inter_mask = diff_id_mask
            R_old_rectified[i][inter_mask] = torch.minimum(R_old[i][inter_mask], sn)

    # L1 归一化每行
    R_old_rectified = R_old_rectified / (R_old_rectified.sum(dim=1, keepdim=True) + 1e-8)

    # KL 蒸馏
    return self.KLDivLoss(torch.log(R_new + 1e-8), R_old_rectified)
```

#### 替换点 2：`continual_train.py` 中的 `get_adaptive_alpha` → 公式(15)

**当前代码**（`continual_train.py:184-200`）：
```python
# 当前：绝对差，可能越界
alpha = float(1 - Difference)  # Difference 可能 > 1
```

**替换为**：点积相似度（公式15）
```python
def get_adaptive_alpha_v2(args, model_new, model_old, all_train_sets, set_index):
    """
    用 LSTKC++ 公式(15)：δ = 1 - mean(dot(R^s_i, R^l_i))
    保证 δ ∈ [0,1]，避免域间隔大时越界
    """
    # 提取特征
    features_new = extract_features_on_init_loader(model_new, init_loader)
    features_old = extract_features_on_init_loader(model_old, init_loader)

    # 计算关系矩阵
    R_new = get_normal_affinity(features_new, Norm=100)  # [n, n]
    R_old = get_normal_affinity(features_old, Norm=100)  # [n, n]

    # 点积相似度（公式15）
    dot_sim = (R_new * R_old).sum(dim=1).mean()  # 逐行点积再平均
    delta = float(1 - dot_sim)
    delta = max(0.0, min(1.0, delta))  # 严格裁剪到 [0,1]
    return delta
```

**影响范围**：仅修改 `trainer.py` 和 `continual_train.py` 中的两个函数，其余模块（TransNet、特征重索引、Voronoi 初始化等）完全保留。

### 3.3 方案二：长短期模型分解方案（完整融合）

**核心改动**：引入 LSTKC++ 的长短期模型分解，同时保留 Bi-C2R 的 TransNet 特征变换。

#### 模型状态管理

```
每个训练步 t 维护：
  Θ_t^s  ← 当前步训练得到的新模型（短期知识）
  Θ_t^l  ← 长期模型（多域平衡知识，由 M*_o 更新）
  Θ_t    ← 推理模型（Θ_t^l 和 Θ_t^s 融合）
  model_trans   ← TransNet（old→new 特征变换，保留自 Bi-C2R）
  model_trans2  ← TransNet（new→old 特征变换，保留自 Bi-C2R）
```

#### 训练流程改动（`continual_train.py`）


#### M*_o 反向纠正模块（新增函数）

```python
def backward_rectification(model_short_old, model_long_old, init_loader):
    """
    Algorithm 1 of LSTKC++
    在 init_loader（当前新数据集的训练集）上搜索最优 α
    """
    best_mAP = 0.0
    best_alpha = 0.0

    for alpha in [i * 0.1 for i in range(11)]:  # 0.0, 0.1, ..., 1.0
        model_fused = linear_combination(args, model_long_old, model_short_old, alpha)
        model_fused.eval()
        with torch.no_grad():
            features, _ = extract_features(model_fused, init_loader)
        # 计算自排序 mAP（features 同时作为 query 和 gallery）
        mAP = compute_self_mAP(features, init_loader.dataset)
        if mAP > best_mAP:
            best_mAP = mAP
            best_alpha = alpha

    model_long_new = linear_combination(args, model_long_old, model_short_old, best_alpha)
    return model_long_new
```

## 七、注意事项

1. **STKR 的向量化实现**：论文中的 for 循环纠正在实际实现时应向量化，避免 Python 循环带来的速度瓶颈。可用 `torch.where` + 广播实现。

2. **M*_o 的计算开销**：每步需要对 init_loader 做 11 次前向推理（α 从 0 到 1，步长 0.1），论文 Table V 显示额外开销约 30%，可接受。

3. **t=2 的特殊处理**：第二个数据集训练时，长短期模型相同（都是 Θ_1），此时 C-STKR 退化为普通 STKR，需要在代码中特判。

4. **特征重索引与长期模型的兼容性**：Bi-C2R 的 trans_feat 使用 model_trans 更新旧 gallery 特征，融合后应使用推理模型 Θ_t（而非 Θ_t^s）来提取和更新特征，保持一致性。

5. **STKR 中的自身对角线处理**：纠正时需排除 i=j 的自身对（对角线），避免将自身相似度纳入阈值计算。
